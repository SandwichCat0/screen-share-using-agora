项目原理

整体流程如下：

1. 屏幕共享

打手端使用 DMA 或双机挂运行游戏。
主机发起 屏幕共享，客户通过远程观看实时画面。

2. 语音与声音转发

客户与考官通过 微信语音沟通。

同时麦克风声音会通过本项目的 虚拟网卡传递到游戏内。

声音路径示例：

三角洲游戏声音 -> VB-OUTPUT -> 麦克风 -> 游戏内
3. 游戏日志同步

客户在 三角洲大厅挂机。

游戏日志会从 打手电脑实时同步到 客户电脑的游戏目录。

注意：

本项目原本包含 fsutil.sys 驱动，用于 后期修改 $MFT FILE_TIME。

但由于现在采用 实时日志传递方案，
该驱动已经 被删除，不再需要。

如果你不了解 $MFT 相关内容，可以直接忽略。

4. 日志中转服务器

日志传递需要 中转服务器，地址需要你自己填写。

你也可以选择：

自行搭建（很多开源中转服务器项目）
或找我帮你一条龙部署

注意：

需要修改 protobuf 头部中的客户硬件信息。
本项目源码中已经展示了实现方式。
